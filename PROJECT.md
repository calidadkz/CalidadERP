# CalidadERP — PROJECT.md

## Objective

Довести CalidadERP до состояния **production-ready торговой ERP** для казахстанского рынка — с полным циклом от предрасчёта партии до получения оплаты, корректным учётом склада в мультивалютной среде и надёжным финансовым модулем.

Система должна обеспечивать: закупки из Китая → таможня/расходы → склад → продажи → отгрузка → финансовая сверка — без ручной работы вне системы.

---

## Current State (по состоянию на 2026-04-06)

- **Ядро системы готово:** все основные модули реализованы (Склад, Закупки, Продажи, Приёмка, Отгрузка, Финансы, Партии, Предрасчёты, Комплекты/Опции)
- **БД:** 49 таблиц в Supabase, все ключевые связи установлены
- **Незакоммиченные изменения:** доработки модуля Bundles/Options (MassAddModal, VariantForm, OptionsEditor)
- **Слабые места:** аналитика/отчётность, тестовое покрытие, производительность при больших объёмах данных

## Milestones

### Milestone 1 — Тестирование и закрытие Bundles & Options
**Цель:** Убедиться, что последние изменения в конфигураторе работают корректно и закоммитить.

**Задачи:**
- [ ] Smoke-test: создание опции → вариант (включая массовое добавление) → привязка к товару → заказ с конфигуратором
- [ ] Закоммитить `MassAddModal.tsx`, `VariantForm.tsx`, `OptionsEditor.tsx`

**Критерий готовности:** конфигуратор стабильно работает в заказах и продажах, все изменения в git.

---

### Milestone 2 — Завершить модуль Партии (Batches)
**Цель:** Реализовать полный жизненный цикл партии — от предрасчёта до закрытия с фиксацией реальных показателей.

Предрасчёт — это моделирование партии. После финализации предрасчёта создаётся Партия, в которой ведётся учёт **фактического** исполнения.

**Задачи:**
- [ ] **Фактические расходы** — ввод реальных расходов по категориям (логистика CN, логистика KZ, таможня, брокер, СВХ, прочее) с привязкой к платежам из выписки (`BatchExpense.paymentId`)
- [ ] **Фактические доходы** — привязка поступлений от клиентов к партии (`BatchExpense` с category `revenue`), сверка с продажами
- [ ] **Документы партии** — загрузка и хранение документов (инвойс, ДТ, CMR и др.), просмотр в `BatchDocumentsTab`
- [ ] **Генерация документов** — формирование упаковочного листа и/или других шаблонных документов на основе данных предрасчёта
- [ ] **Экономика партии (план vs факт)** — доработка `BatchEconomyTab`: детальное сравнение плановых показателей из предрасчёта с фактическими расходами и доходами
- [ ] **Закрытие партии** — статус `completed/closed`, финальный расчёт прибыли через `BatchCalculator`

**Критерий готовности:** полный цикл Предрасчёт → Партия → Фактические данные → Закрытие работает без ручной работы вне системы.

### Milestone 3 — Базовая отчётность
**Цель:** Дать руководителю финансовые отчёты и инструменты анализа ассортимента.

**Задачи:**

**Финансовые отчёты:**
- [ ] **ОДДС (Отчёт о движении денежных средств)** — по статьям ДДС (Operating/Investing/Financial) за период, с разбивкой по счетам и валютам
- [ ] **ОПиУ (Отчёт о прибылях и убытках)** — доходы и расходы с разбивкой: постоянные / переменные / проектные расходы
- [ ] **Отчёт по эффективности Партий** — сводная таблица: план/факт прибыли, отклонение, маржинальность по каждой партии

**Анализ ассортимента:**
- [ ] **ABC-XYZ анализ** — ABC по выручке/прибыли (A=80%, B=15%, C=5%), XYZ по стабильности спроса (коэффициент вариации); итоговая матрица AX/AY/AZ/BX... для принятия решений по закупкам

**Критерий готовности:** все отчёты доступны через навигацию, поддерживают выбор периода, данные можно экспортировать в Excel.
