# Модуль: Права доступа (Permissions Manager)

Инструмент глобального управления безопасностью и видимостью данных. Модуль реализует модель **PBAC (Permission-Based Access Control)**, где права настраиваются гранулярно для каждой функциональной роли на основе динамической карты системы.

---

## 1. Основной экран: Матрица управления

Интерфейс разделен на две основные зоны для быстрой навигации и настройки.

### Шапка управления (Header)
*   **Заголовок:** «Права доступа» с иконкой `Shield` цвета индиго.
*   **Инструменты:**
    *   **Селектор ролей:** Переключатель между ролями (Менеджер, Бухгалтер, Снабженец и т.д.).
    *   **Экспорт/Импорт:** Синхронизация всей матрицы прав через CSV. Позволяет быстро тиражировать настройки безопасности между филиалами.
    *   **Сохранить:** Акцентная кнопка (`bg-blue-600`), фиксирующая изменения в таблице `role_permissions`.

---

## 2. Динамический реестр и Уровни доступа

Модуль строится на базе «Карты Системы» (`SystemRegistry.ts`). Любой новый объект, добавленный в код, автоматически появляется в интерфейсе.

### Уровни доступа (AccessLevel):
1.  **Скрыто (None):** Объект полностью удаляется из UI (скрывается колонка, кнопка или вкладка меню).
2.  **Просмотр (Read):** Данные видны, но поля ввода заблокированы (read-only), а кнопки действий неактивны.
3.  **Правка (Write):** Полный доступ к изменению данных и выполнению операций.

### Структура настройки (Правая панель):
*   **Вкладки и Режимы:** Контроль доступа к целым разделам и табам внутри модулей.
*   **Поля и Данные:** Управление видимостью конкретных колонок (напр. «Цена закупа») или финансовых KPI.
*   **Действия:** Разрешение на выполнение критических операций (напр. «Сторно», «Проведение», «Импорт»).

---

## 3. Бизнес-логика: Безопасность данных

*   **Приоритет Админа:** Учетная запись с ролью `admin` всегда имеет уровень `write` на все объекты, игнорируя настройки матрицы.
*   **Мгновенное применение:** Права кешируются в `sessionStorage` при входе. Для применения глобальных изменений матрицы пользователю может потребоваться обновить страницу или перезайти в систему.
*   **Защита от отключения:** Если доступ к разделу выключен главным чекбоксом в левой панели, все вложенные элементы (поля/кнопки) автоматически блокируются оверлеем «Раздел выключен».

---

## 4. Визуальные стандарты UI
*   **Стиль «Светофор»:** 
    *   Красный (`text-red-600`): Скрыто.
    *   Синий (`text-blue-600`): Только чтение.
    *   Зеленый (`text-emerald-600`): Полный доступ.
*   **Интерактив:** Использование `backdrop-blur` и иконки `Lock` для визуализации закрытых разделов.
*   **Состояния:** Пульсация кнопки «Сохранить» при наличии несохраненных изменений (`isDirty`).
