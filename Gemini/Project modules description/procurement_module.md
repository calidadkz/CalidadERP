# Модуль: Снабжение (Procurement / Supplier Orders)

Центральный узел управления закупками ТМЦ у внешних контрагентов. Модуль обеспечивает связь между потребностью в товаре, размещением заказа (ЗП) и финансовым планированием оплат производителям.

---

## 1. Основной экран: Журнал заказов

Реестр всех документов закупа, выполненный в стиле «Enterprise Data Grid».

### Шапка управления (Header)
*   **Заголовок:** «Заказы поставщикам» с иконкой `ShoppingCart` синего цвета.
*   **Инструменты:**
    *   **Фильтр по поставщику:** Выпадающий список для быстрой фильтрации журнала.
    *   **Новый заказ (ЗП):** Акцентная кнопка (`bg-blue-600`), открывающая интерактивную форму создания.

### Таблица документов:
*   **ID / Дата:** Бизнес-идентификатор (напр., `#AF32D1`) синим моноширинным шрифтом и дата документа.
*   **Поставщик:** Наименование контрагента (жирный шрифт).
*   **Сумма (ВПл):** Итоговая стоимость в валюте инвойса (USD, CNY и др.).
*   **Оплата (IPP):** Визуальный прогресс-бар, показывающий степень покрытия суммы заказа фактически исполненными платежами.
*   **Статус:** Цветной бейдж стадии обработки (`Подтвержден`, `Частично получен`, `Закрыт`).

---

## 2. Форма заказа: Двухуровневый интерфейс

Полноэкранный редактор с разделением на товарную и финансовую части через Segmented Control.

### Вкладка «1. Состав инвойса»
Рабочая область для формирования спецификации закупа.
*   **Умный подбор:** Поиск по артикулу или названию. При выборе станка автоматически открывается **ConfiguratorModal**.
*   **Индикация дефицита:** При подборе товара система подсвечивает красным бейджем позиции, по которым на складе зафиксирован отрицательный свободный остаток (дефицит).
*   **Мультивалютность:** Каждая строка содержит автоматический расчет кросс-курса: `(Валюта товара -> Валюта инвойса)`.

### Вкладка «2. График оплат (IPP)»
Инструмент формирования платежного календаря.
*   **Контроль лимитов:** Блок вверху вкладки показывает «Остаток графика». Подсвечивается красным, если сумма запланированных траншей превышает сумму инвойса.
*   **Транши:** Список дат и сумм будущих оплат. Результат сохраняется в глобальный Календарь платежей.

---

## 3. Интеграция с Конфигуратором

При добавлении станка в заказ снабжения используется специальный режим **Procurement Mode**:
*   **Фокус:** Заводская цена (ВЦП) и выбор технических опций.
*   **Логика:** Выбор опций изменяет итоговую цену закупа в валюте производителя.
*   **Результат:** В заказ передается не просто "Станок", а массив выбранных опций, который сохраняется в `configuration` и будет использован при приемке на склад.

---

## 4. Бизнес-логика и Складской учет
*   **Событие «Confirmed»:** При сохранении нового заказа система автоматически создает движения типа `In Incoming`. Товар появляется в колонке «В пути» модуля Остатков.
*   **Фиксация курса:** При создании заказа фиксируется «Провизорный курс» для оценки будущих затрат в KZT.
*   **Связь с приемкой:** Заказ является единственным основанием для создания документа «Приемка на склад».

---

## 5. Визуальные стандарты UI
*   **Контрастность:** Итоговая сумма заказа отображается шрифтом `text-4xl font-black` на светлом фоне для максимальной видимости.
*   **Реактивность:** Пересчет итогов инвойса и кросс-курсов происходит мгновенно при изменении любого поля в таблице товаров.
*   **Безопасность:** Кнопка «Провести» блокируется, если не выбран поставщик или сумма графика оплат не совпадает с суммой заказа.
