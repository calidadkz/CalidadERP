
# Модуль: Банковские выписки (IP - Actual Payments)

Узел фиксации фактических денежных потоков и их сопоставления с плановыми ожиданиями. Модуль обеспечивает юридическую и финансовую чистоту взаиморасчетов с контрагентами.

---

## 1. Основной экран: Журнал выписок (IP)

Реестр всех подтвержденных банковских транзакций. **Критическое отличие:** Выписки не имеют собственной статьи ДДС. Аналитика расходов и доходов подтягивается через привязанные Плановые платежи (IPP).

### Шапка управления (Header)
*   **Заголовок:** «Выписки (IP)» с иконкой `Receipt` цвета индиго.
*   **Инструменты:**
    *   **Живой поиск:** Фильтрация по названию контрагента или номеру транзакции.
    *   **Быстрые кнопки:** Создание исходящего платежа (Красная) или фиксация входящей оплаты (Зеленая).

### Таблица фактических платежей:
*   **Дата / ID:** Дата проведения операции и системный номер транзакции (в стиле `TX-XXXX`).
*   **Контрагент:** Название компании (Поставщик/Клиент).
*   **Счет:** Указание расчетного счета (Банк + Номер), через который прошли средства.
*   **Сумма:** Цветовая индикация направления (Красный — расход, Зеленый — доход).
*   **Статус разноски:** 
    *   `Разнесено` (Изумрудный бейдж): Сумма полностью привязана к планам.
    *   `Разнести [Сумма]` (Оранжевая кнопка): Индикатор наличия свободного остатка, требующего распределения.

---

## 2. Бизнес-логика: Умная разноска (Allocation)

Механизм связи «многие-ко-многим», позволяющий одной выпиской закрывать несколько плановых траншей.

### Процесс распределения:
*   При клике на кнопку «Разнести» открывается модальное окно с перечнем всех открытых (неоплаченных) траншей IPP по данному контрагенту.
*   Каждый транш IPP имеет свою статью ДДС (напр. «Оплата за товар», «Аренда»).
*   После распределения суммы выписки между планами, выписка косвенно приобретает аналитическую окраску.

---

## 3. Визуальные стандарты UI
*   **Иерархия (Accordion):** Строки журнала расширяемые. При клике на Chevron раскрывается детализация: на какие именно заказы (ЗП/ЗК) и под какие Статьи ДДС была распределена сумма данного платежа.
*   **Интерактивность:** Использование `backdrop-blur` для модального окна разноски, чтобы фокусировать внимание бухгалтера на финансовых расчетах.
*   **Типографика:** Использование `font-mono` для всех денежных сумм.
