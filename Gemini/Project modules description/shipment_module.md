# Модуль: Отгрузка (Shipment / Накладные)

Финальное звено цепочки продаж, отвечающее за физический отпуск товаров со склада и снятие резервов.

---

## 1. Основной экран: Очередь и Архив

Интерфейс разделен на оперативную зону (отгрузка в процессе) и исторический журнал.

### Блок «Ожидают отгрузки» (Pending Orders)
Сетка карточек заказов клиентов (ЗК), которые еще не были полностью отгружены.
*   **Метрики:** Прогресс в штуках (напр., `2 / 5 шт.`) и общая ценность отгружаемой партии.
*   **Действие:** Клик по карточке открывает форму накладной, предзаполненную остатками по заказу.

### Блок «Журнал отгрузок» (Journal)
Реестр всех расходных накладных.
*   **Статусы:** `Черновик` (Серый), `Проведено` (Изумрудный).
*   **Действия:** Для проведенных документов доступна кнопка **«Сторно»** (RotateCcw), возвращающая товар на склад.

---

## 2. Форма отгрузки: Складской контроль

Рабочая область для фиксации факта передачи товара клиенту.

### Товарная сетка (Items Table)
*   **Складской контроль:** Колонка «На складе» показывает реальное наличие конкретной комплектации (Base + Options). 
*   **Валидация:** Если «Отгрузить» > «На складе», поле подсвечивается красным, а проведение блокируется через оверлей «Ошибка склада».
*   **Комплектации:** Для станков отображается полный список установленных опций в виде компактных синих тегов.

### Финансовое резюме (Right Panel)
Контрастная темная панель (`bg-slate-900`) для визуального контроля ценности отпуска.
*   **Сумма к списанию:** Итоговая стоимость накладной в KZT.
*   **Безопасность:** Если у пользователя нет прав `col_price`, финансовые блоки скрываются (заменяются на «Цены скрыты»).

---

## 3. Бизнес-логика и Складской учет
*   **Событие «Posted» (Проведение):** Система генерирует парные движения:
    1.  `Out Physical`: Товар списывается с остатков склада.
    2.  `Out Reserved`: Соответствующее количество списывается из регистра резервов заказа.
*   **Механизм Сторно:** При отмене проведения все движения по накладной удаляются из реестра `StockMovements`, возвращая товар в статус «В наличии» и «Зарезервировано».
*   **Частичная отгрузка:** Один заказ (ЗК) может быть отгружен несколькими накладными.

---

## 4. Визуальные стандарты UI
*   **Иконографика:** `Truck` (Логистика), `PackageMinus` (Списание), `RotateCcw` (Возврат).
*   **Цветовые коды:** Оранжевый — для операций отгрузки, Красный — для дефицита, Изумрудный — для успешного завершения.
*   **Типографика:** Использование `font-mono` для всех ID документов и складских ячеек.
