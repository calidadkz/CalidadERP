
# Модуль: Номенклатура (Nomenclature)

Центральный хаб управления товарным каталогом. Модуль отвечает за создание, классификацию и финансовое планирование всех ТМЦ (товаров, станков и услуг), используемых в системе.

---

## 1. Основной экран: Журнал номенклатуры

Главный реестр товаров, выполненный в стиле "Enterprise Data Grid".

### Шапка управления (Header)
*   **Заголовок:** «Номенклатура» с иконкой `Box` синего цвета.
*   **Инструменты:**
    *   `Экспорт`: Кнопка скачивания всего каталога в формате CSV с сохранением кодировки UTF-8 (BOM).
    *   `Импорт`: Загрузка данных из CSV. Содержит систему валидации: проверяет существование категорий, поставщиков и типов валют. При ошибках выводит детальный лог во всплывающем окне.
    *   `Добавить товар`: Акцентная синяя кнопка (`bg-blue-600`), открывающая Wizard-модалку создания.
*   **Поиск:** Интерактивное поле под заголовками таблицы. Позволяет искать одновременно по части названия или артикулу (SKU).

### Таблица товаров
*   **Типизация (Иконки):** 
    *   `Box` (Синий) — Станки.
    *   `Zap` (Оранжевый) — Запчасти.
    *   `Briefcase` (Фиолетовый) — Услуги.
*   **Колонки:**
    *   `Артикул / Название`: Кликабельное название для перехода в режим редактирования. SKU отображается мелким моноширинным шрифтом под названием.
    *   `Цена (ВЦП)`: Закупочная стоимость в валюте поставщика. Скрывается настройками прав доступа.
    *   `Продажная`: Итоговая цена в KZT. Выделена жирным шрифтом на светло-зеленом фоне (`bg-green-50`).
    *   `Габариты / Объем`: Формула `Д x Ш x В` и автоматически вычисленный объем в м³ (жирным).
*   **Действия:** Кнопки «Карандаш» (Edit) и «Корзина» (Move to Trash). При удалении используется оверлей подтверждения.

## 2. Карточка товара (Product Wizard)

Полноэкранное модальное окно (`h-[90vh]`) с системой вкладок.

### Шапка модального окна
*   Темный фон (`bg-slate-800`), отображает SKU и текущий режим (Просмотр/Редактирование).
*   Индикатор «Режим просмотра» (`Eye`), если у пользователя нет прав `write` на модуль.

### Вкладка 1: Общие данные (General)
Разделена на три логических блока:

1.  **Основная информация:**
    *   `Тип`: Выбор из Machine/Part/Service. Меняет доступность других вкладок.
    *   `Категория`: Зависимый список (фильтруется по типу товара).
    *   `Поставщик`: Привязка к контрагенту для фильтрации в заказах (ЗП).
    *   `Наименование`: Крупное текстовое поле для полного торгового названия.

2.  **Логистика и Габариты:**
    *   Поля ввода `Д/Ш/В` в миллиметрах.
    *   `Объем (авто)`: Неинтерактивное поле, пересчитывается мгновенно при вводе габаритов.
    *   `Вес (кг)`: Поле на оранжевом фоне, критично для расчета стоимости доставки в будущем.

3.  **Ценообразование:**
    *   `Цена ВЦП (Закуп)`: Поле ввода + выбор валюты (USD, CNY, EUR, RUB, KZT).
    *   `Наценка %`: Ручной ввод (если профиль не назначен) или заблокированное поле (если работает Smart Pricing).
    *   `Цена продажи (KZT)`: Итоговый расчет. При использовании профиля подсвечивается синим (`bg-blue-50`).
    *   **Кнопка «Детализация расходов»:** Доступна только при активном профиле ценообразования.

---

## 3. Боковая панель: Детализация экономики (Unit Economics Drawer)

Выдвижной оверлей справа (`w-[360px]`), открывающийся из вкладки «Общие данные». 

### Состав панели:
*   **Профиль:** Название примененного правила (напр. "Лазеры JK").
*   **Секция «Расходы»:** Список всех затрат, добавленных к цене закупа:
    *   Доставка (Китай и Локальная), СВХ, Брокер, Пусконаладка (ПНР), Доставка до клиента, Бонус отдела продаж.
*   **Секция «Налоги»:** Расчетные суммы НДС (16%) и КПН (20%), которые будут удержаны из выручки.
*   **Итоги:** 
    *   `Себестоимость`: Сумма всех затрат (серая плашка).
    *   `Чистая прибыль`: Прогнозируемый доход в тенге (зеленая плашка).
    *   `Продажная цена`: Финальный результат на синем фоне.

---

## 4. Вкладка 2: Опции и Варианты (Только для станков)

Инструмент маппинга конфигуратора.

### Сплит-интерфейс:
*   **Слева (Список групп):** Перечень всех типов опций (Чиллеры, Трубки и т.д.).
    *   Чекбоксы включения группы для данной модели.
    *   Кнопка быстрого создания нового типа опции (`+ Тип`).
*   **Справа (Таблица вариантов):** Список конкретных запчастей, входящих в выбранную группу.
    *   `Default`: Выбор варианта, который будет стоять в конфигураторе "по умолчанию".
    *   `Спец. цена (KZT)`: Поле для переопределения цены запчасти именно для этой модели станка (Price Override). Если заполнено, подсвечивается тегом `SPEC`.

---

## 5. Вкладка 3: Внутренний состав / BOM (Только для станков)

Редактор базовой спецификации (из чего состоит станок без опций).

### Элементы:
*   **Панель подбора:** Выпадающий список всех товаров типа «Запчасть» (`PART`) с поиском по SKU.
*   **Таблица состава:** 
    *   Список запчастей с возможностью редактирования количества прямо в строке.
    *   Автоматическая склейка: при добавлении уже существующей позиции её количество суммируется.
    *   Кнопка «Удалить из состава» (Trash2).

---

## 6. Технические особенности UI
*   **Анимация:** Переходы между вкладками используют `animate-in fade-in`. Боковая панель экономики выезжает с плавным `duration-300`.
*   **Валидация:** Кнопка «Сохранить изменения» блокируется, если не заполнены обязательные поля (Наименование, SKU).
*   **Реактивность:** Любое изменение цены закупа или параметров в профиле мгновенно обновляет все цифры во всех вкладах и боковой панели без перезагрузки страницы.
