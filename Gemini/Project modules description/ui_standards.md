# Стандарты Интерфейса ERP (UI Standards)

Для обеспечения визуальной целостности и удобства пользователя во всех функциональных модулях системы применяется единый стандарт верстки.

## 1. Глобальный контейнер
Каждый основной вид (View) должен быть обернут в контейнер с вертикальным шагом:
```tsx
<div className="space-y-6">
   {/* Содержимое */}
</div>
```
*ВАЖНО:* Скрытые элементы (инпуты для файлов, порталы уведомлений) должны располагаться ВНЕ этого контейнера или во фрагменте `<>`, чтобы не нарушать работу селектора `space-y` (который добавляет верхний отступ всем элементам, кроме первого).

## 2. Заголовок раздела (Main Header)
Заголовок всегда является ПЕРВЫМ элементом в `space-y-6`.

### Структура заголовка:
```tsx
<div className="flex justify-between items-center">
    <div>
        <h2 className="text-2xl font-bold text-slate-800 flex items-center">
            <Icon className="mr-3 text-accent-600" size={28} /> Название раздела
        </h2>
        <p className="text-slate-500 text-sm font-medium mt-1">Краткое описание назначения модуля</p>
    </div>
    <div className="flex items-center gap-3">
        {/* Кнопки действий */}
    </div>
</div>
```

### Параметры:
- **Шрифт заголовка:** `text-2xl font-bold text-slate-800`. Обычный регистр (не uppercase).
- **Иконка:** Размер `28`, отступ справа `mr-3`, цвет акцентный (обычно `text-blue-600` или `text-indigo-600`).
- **Подзаголовок:** `text-slate-500 text-sm font-medium mt-1`. Обычный регистр.

## 3. Цветовое кодирование иконок
- **Синий (`text-blue-600`):** Склад, Номенклатура, Снабжение, Продажи, Приемка.
- **Индиго (`text-indigo-600`):** Опции, Категории, Финансы.
- **Оранжевый (`text-orange-600`):** Запчасти, Отгрузка.
- **Красный (`text-red-600`):** Брак, Удаление.

## 4. Формы документов
Для форм создания (ЗК, ЗП, Приемка, Отгрузка) используется тот же стандарт заголовка, что и в журналах, чтобы переход между списком и формой не вызывал «скачка» интерфейса.
