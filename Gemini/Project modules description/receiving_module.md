# Модуль: Приемка на склад (Receiving / Warehouse Entry)

Финальный узел цепочки снабжения, отвечающий за оприходование товаров и расчет их финальной себестоимости (Landed Cost). Модуль обеспечивает переход ТМЦ из статуса «В пути» в физический остаток на складе с учетом всех сопутствующих затрат.

---

## 1. Основной экран: Журнал и Очередь

Интерфейс разделен на оперативную зону и архив документов.

### Блок «Ожидают поступления» (Pending Orders)
Сетка карточек заказов поставщикам (ЗП), которые еще не были полностью оприходованы.
*   **Визуал:** Сетчатый макет (`Grid`) с белым фоном и синими акцентами.
*   **Метрики:** Количество позиций в заказе, общая сумма инвойса и индикатор «В пути».
*   **Действие:** Клик по карточке открывает форму приемки, предзаполненную данными из заказа.

### Блок «Журнал поступлений» (Journal)
Реестр проведенных и черновиковых документов приемки.
*   **Колонки:** ID приемки, Дата, Ссылка на ЗП, Склад зачисления, Сумма доп. расходов и Итоговая ценность партии в KZT.
*   **Статусы:** `Черновик` (Серый), `Оприходовано` (Изумрудный).

---

## 2. Форма приемки: Расчет себестоимости

Рабочая область для детального распределения затрат и фиксации фактического прихода.

### Товарная сетка (Items Table)
*   **Сравнение Qty:** Колонки «План» (из заказа) и «Факт» (ввод вручную).
*   **Финансы:** Отображение «Базовой цены» в KZT (пересчитанной по FIFO) и колонки «Доп. Расходы» (результат распределения).
*   **Итог:** Колонка «Итого ед.», показывающая полную себестоимость одной единицы товара.

### Блок «Дополнительные расходы» (Expenses)
Инструмент для добавления затрат на логистику, таможню и брокеров.
*   **Типы распределения:** 
    *   `По стоимости`: Пропорционально цене товаров.
    *   `По объему`: Пропорционально м³ (критично для доставки).
    *   `По количеству`: Равномерно на каждую штуку.
    *   `На позицию`: Точечное назначение расхода на конкретный SKU.

---

## 3. Аналитика: Экономика приемки (Right Panel)

Контрастная панель для контроля финансовых показателей документа.
*   **Эффективный курс:** Расчетный курс KZT, полученный по методу FIFO на основе фактических оплат по этому заказу.
*   **Итоговая ценность:** Крупный блок `text-3xl font-black`, отображающий суммарную стоимость всей партии, которая встанет на баланс предприятия.

---

## 4. Бизнес-логика и Складской учет
*   **Событие «Posted»:** При проведении система генерирует движения:
    1.  `In Physical`: Товар появляется на остатках склада.
    2.  `Out Incoming`: Соответствующее количество списывается из колонки «В пути».
*   **Статус заказа:** После проведения приемки статус связанного заказа (ЗП) автоматически меняется на `Частично получен` или `Закрыт`.
*   **Брак и Несоответствия:** Если «Факт» меньше «Плана», система предлагает создать запись в модуле «Брак и потери».

---

## 5. Визуальные стандарты UI
*   **Иконографика:** `PackageCheck` (Приемка), `Truck` (Логистика), `Calculator` (Расчеты).
*   **Цветовые коды:** Синий — для активных операций, Оранжевый — для расходов, Изумрудный — для успешного оприходования.
*   **Безопасность:** Кнопка «Оприходовать» требует обязательного выбора склада и подтверждения фактического количества.
