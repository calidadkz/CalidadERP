
# Модуль: Банковские выписки (IP - Actual Payments)

## 1. Структура данных и Функционал
Регистрация фактов движения денег и механизм связи "многие-ко-многим", реализованный в `FinanceStatements.tsx`.

### Сущность (ActualPayment)
*   **allocations:** Коллекция связей `PaymentAllocation`. Один платеж может закрывать несколько траншей IPP.
*   **exchangeRate:** Фиксируется на момент проведения. Для валюты закупа используется расчет по FIFO.

### Бизнес-логика: Умная разноска
*   **Механизм:** Если выписка не привязана к плану (или привязана частично), появляется кнопка «Разнести [Сумма]».
*   **Алгоритм распределения:** Пользователь выбирает один или несколько открытых траншей IPP чекбоксами. Система автоматически распределяет остаток суммы выписки между ними, начиная с самого раннего.

---

## 2. Дизайн и Интерфейс (UI/UX)

### Журнал выписок
*   **Иерархия:** Строки таблицы расширяемые (Accordion). При клике на `Chevron` раскрывается список всех заказов, которые закрыл этот платеж.
*   **Цветовая схема:** Отрицательные суммы (Outgoing) — красный `text-red-600`, положительные (Incoming) — зеленый `text-emerald-600`.

### Модальное окно разноски
*   **Split-индикатор:** В верхней части окна два блока. Слева — полная сумма выписки, справа — динамически меняющийся остаток "Доступно для распределения".
*   **Список счетов:** Карточки IPP с чекбоксами. При выборе карточки она подсвечивается синим (`bg-blue-50`), а текст подсказывает: "Закроется полностью" или "Закроется частично".
