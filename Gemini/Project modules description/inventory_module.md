# Модуль: Остатки и Движения (Inventory)

Аналитическое ядро системы, обеспечивающее прозрачный учет товарно-материальных ценностей на всех этапах жизненного цикла. Модуль реализован на базе событийно-ориентированной модели (Registry), где текущий баланс является производной от суммы всех складских транзакций.

---

## 1. Основной экран: Глобальная навигация

Интерфейс разделен на два режима просмотра, переключение между которыми осуществляется через компактный Segmented Control в шапке модуля.

### Шапка управления (Header)
*   **Заголовок:** «Остатки и Движения» с иконкой `PackageSearch` синего цвета.
*   **Инструменты:**
    *   `Поиск`: Живой фильтр по названию модели или артикулу (SKU).
    *   `Ввод остатков`: Кнопка вызова формы ручной корректировки (доступна только при наличии прав `write`).
*   **Режимы:**
    *   **Остатки:** Агрегированная ведомость наличия.
    *   **Движения:** Детальный журнал всех операций (логов) склада.

---

## 2. Экран «Остатки»: Аналитика и Иерархия

Центральная рабочая область для контроля складских запасов.

### Информационные блоки (KPI)
*   **Ценность склада:** Сумма произведений `(Физический остаток * Средневзвешенная себестоимость)` по всем позициям. Показывает объем "замороженного" капитала в закупке.
*   **Потенциальная выручка:** Сумма произведений `(Физический остаток * Актуальная цена продажи)`. Цена продажи для станков рассчитывается динамически с учетом установленных в комплектации опций.

### Иерархическая таблица (Accordion)
Использует логику группировки «Модель -> Комплектация»:
*   **Уровень 1 (Модель):** Отображает агрегированные данные по базовой модели станка или запчасти. Суммирует остатки по всем возможным сборкам.
*   **Уровень 2 (Комплектации):** Разворачивается при клике на модель. Каждая строка — это уникальный товар (SKU), имеющий свой набор опций, свою себестоимость и цену продажи.
*   **Статусы остатков:**
    *   `Склад`: Физическое наличие на полках.
    *   `Путь`: Ожидаемые поступления по подтвержденным заказам (ЗП).
    *   `Резерв`: Товар, забронированный под заказы клиентов (ЗК).
    *   `Свободно`: Расчетный остаток `(Склад + Путь - Резерв)`. Отрицательные значения подсвечиваются красным (сигнал дефицита).

---

## 3. Инструмент: Ввод остатков (Stock Adjustment)

Форма для первичного наполнения склада или проведения инвентаризации.

### Особенности процесса:
*   **Для запчастей:** Прямой ввод количества и себестоимости единицы.
*   **Для станков:** Требуется обязательный выбор конфигурации (опций) через встроенный мини-конфигуратор. Система автоматически рассчитывает рекомендуемую себестоимость и цену продажи на основе активного Профиля цен, но оставляет возможность ручной правки.
*   **Результат:** Создает движение типа `Adjustment`, которое фиксируется в истории.

---

## 4. Экран «Движения»: Журнал транзакций

Реестр всех событий, изменивших состояние склада.

### Состав записи:
*   **Тип операции:** Приход (`In`) — зеленый цвет, Расход (`Out`) — красный цвет.
*   **Статус ТМЦ:** Физический склад, Путь (Incoming) или Резерв.
*   **Аналитика:** Ссылка на документ-основание (Заказ, Приемка, Накладная) и краткое описание (напр. "Списание брака").
*   **Финансы:** Фиксация себестоимости единицы в KZT на момент совершения операции.

---

## 5. Технические особенности UI
*   **Авто-скрытие:** Позиции с нулевыми остатками по всем регистрам (Склад/Путь/Резерв) автоматически скрываются из таблицы Остатков для разгрузки интерфейса.
*   **Реактивность:** Пересчет "Свободного остатка" и KPI происходит мгновенно при поступлении данных из `InventoryMediator`.
*   **Визуальный контроль:** Колонки с ценами и суммами выручки скрываются автоматически, если у пользователя нет соответствующих прав доступа в матрице разрешений.
*   **Сортировка:** Поддержка многоуровневой сортировки (по типу, алфавиту или объему запасов).
