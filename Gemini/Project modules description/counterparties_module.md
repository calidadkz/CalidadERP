# Модуль: Контрагенты (Counterparties)

## 1. Структура данных и Функционал
Единый реестр внешних организаций (Поставщиков и Клиентов).

### Поля Поставщика (Supplier)
*   **name:** Название компании.
*   **country:** Страна (важно для расчета логистических плеч).
*   **currency_default:** Валюта, в которой обычно выставляются инвойсы.

### Поля Клиента (Client)
*   **name:** Юридическое или торговое название.
*   **contact_person:** ФИО основного менеджера.
*   **phone:** Контактный номер (форматированный).
*   **credit_limit:** Лимит дебиторской задолженности (для блокировки отгрузок).

### Бизнес-логика
*   **Синхронизация:** При создании контрагента можно указать обе роли. Система создаст две записи с общим бизнес-именем, но разными профилями ответственности.
*   **Аналитика:** В карточке контрагента отображается сумма всех "Планируемых платежей" (IPP), позволяя видеть будущий денежный поток по конкретному партнеру.
*   **Удаление:** Невозможно удалить контрагента, по которому есть ненулевое сальдо в "Выписках" (IP).

## 2. Дизайн и Интерфейс (UI/UX)

### Основной экран
*   **Переключатель ролей:** Крупные кнопки в стиле "Segmented Control" сверху (Поставщики / Клиенты).
*   **Поиск:** Глобальная строка поиска по названию, телефону или стране.
*   **Таблица:**
    *   **Строка:** Содержит название компании и краткие теги (напр., "Китай", "VIP", "Долг: 1.2M").
    *   **Цветовая индикация:** Контрагенты с просроченными платежами подсвечиваются мягким красным фоном.

### Боковая панель (Side Drawer)
Создание и редактирование происходит в выдвижной панели справа, чтобы не терять контекст списка.
*   **Группировка:** Поля разделены на "Основное", "Роли" и "Контакты".
*   **Интерактив:** При переключении тумблера "Является Клиентом" динамически появляются поля для ввода телефона и ФИО.

### Карточка контрагента (Detail View)
*   **Финансовый дашборд:** Мини-график оплат за последние 6 месяцев.
*   **Список документов:** Быстрый доступ к связанным заказам и отгрузкам прямо из карточки.
