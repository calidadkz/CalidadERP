
# Модуль: Продажи (Sales Orders / Заказы клиентов)

Центральный узел управления коммерческими отношениями с клиентами. Модуль обеспечивает полный цикл от подбора конфигурации оборудования до контроля графиков оплат и резервирования складских остатков.

---

## 1. Основной экран: Журнал заказов (ЗК)

Реестр всех документов продаж, выполненный в стиле «Enterprise Data Grid».

### Шапка управления (Header)
*   **Заголовок:** «Продажи (ЗК)» с иконкой `ShoppingCart` синего цвета.
*   **Инструменты:**
    *   **Поиск по клиенту:** Живой фильтр по названию компании или ФИО.
    *   **Новый заказ (ЗК):** Акцентная кнопка (`bg-blue-600`), открывающая форму создания.

### Таблица документов:
*   **Бизнес ID / Дата:** Короткий номер (напр., `#AF32D1`) синим моноширинным шрифтом и дата создания.
*   **Клиент:** Название компании-клиента (жирным шрифтом).
*   **Сумма:** Итоговая стоимость в KZT.
*   **Оплата (IPP):** Визуальный прогресс-бар (Синий -> Изумрудный), показывающий покрытие суммы заказа фактически исполненными платежами.
*   **Отгрузка (Статус):** Текстовый индикатор (`Не отгружен`, `Частично`, `Отгружен`).

---

## 2. Форма заказа: Компактный интерфейс

Полноэкранный редактор с разделением на товарную и финансовую части через Segmented Control.

### Верхняя панель данных (Header Summary)
Компактная строка с основными реквизитами:
*   **Клиент:** Выпадающий список с кнопкой быстрого создания (`UserPlus`).
*   **Валюта:** Информационный блок с указанием базовой валюты (KZT).
*   **Итого заказ:** Крупный текстовый блок (`text-2xl font-black`) для визуального контроля суммы.

### Вкладка «1. Состав заказа»
*   **Умный подбор:** Поиск по артикулу или названию в синем контейнере `bg-blue-50/20`. Селектор категории и строка поиска расположены в один ряд.
*   **Конфигуратор станков:** При выборе станка открывается **ConfiguratorModal** в режиме **Sales Mode**.
*   **Индикация остатков:** В списке подбора отображается `Доступно: [X]`. Дефицит подсвечивается красным бейджем.
*   **Таблица состава:** Оптимизирована для высокой плотности данных. Использует моноширинные шрифты для цен и артикулов.

### Вкладка «2. График оплат (IPP)»
*   **Контроль распределения:** Блок «Остаток к распределению» показывает разницу между суммой заказа и суммой траншей. 
*   **Транши:** Список будущих платежей с компактными полями даты и суммы.

---

## 3. Бизнес-логика: Резервирование
*   **Событие «Confirmed»:** При сохранении/проведении заказа система автоматически создает движения типа `In Reserved`. 
*   **Логика:** Товар физически остается на складе, но его статус «Свободно» уменьшается. Это предотвращает продажу одного и того же станка двум клиентам одновременно.

---

## 4. Визуальные стандарты UI
*   **Контрастность:** Итоговая сумма заказа выделена синим цветом `text-blue-600`.
*   **Быстрый клиент:** Иконка `UserPlus` позволяет создать нового контрагента, не закрывая форму.
*   **Безопасность:** Кнопка «Провести» блокируется, если сумма графика оплат не совпадает с суммой состава заказа.
