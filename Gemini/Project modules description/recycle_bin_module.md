# Модуль: Корзина (Recycle Bin)

Система защиты данных от случайного удаления. Модуль реализует механизм "мягкого удаления" (Soft Delete), позволяя восстанавливать объекты с сохранением всех связей.

---

## 1. Основной экран: Журнал удалений

Реестр объектов, временно исключенных из активной бизнес-логики.

### Шапка управления (Header)
*   **Заголовок:** «Корзина системы» с иконкой `Trash2` красного цвета.
*   **Инструменты:** Информационная плашка `AlertTriangle` с предупреждением о деструктивности окончательного удаления.

### Таблица объектов:
*   **Тип объекта:** Классификация (Product, Category, Bundle и т.д.).
*   **Наименование / ID:** Имя объекта и его `original_id` (бизнес-ключ).
*   **Дата удаления:** Когда объект был перемещен в корзину.
*   **Действия:** 
    *   `Вернуть`: Кнопка восстановления объекта в исходный справочник.
    *   `Стереть`: Кнопка окончательного (физического) удаления из БД.

---

## 2. Бизнес-логика: Восстановление и Очистка

*   **Механизм восстановления (Restore):** 
    1. Система извлекает JSON-данные из записи в корзине.
    2. Восстанавливает запись в оригинальную таблицу (напр. `products`) с сохранением старого ID.
    3. Удаляет запись из таблицы `trash`.
    4. Генерирует лог `Create` в истории.
*   **Окончательное удаление (Purge):** Безвозвратное удаление записи из таблицы `trash` по системному UUID. 
*   **Изоляция:** Объекты, находящиеся в корзине, полностью игнорируются всеми алгоритмами системы (не учитываются в остатках, не видны в поиске заказов).

---

## 3. Визуальные стандарты UI
*   **Безопасность:** Окончательное удаление требует подтверждения через **Inline Confirmation** (появление кнопок "ДА/НЕТ" прямо в строке), чтобы исключить случайную потерю данных.
*   **Цветовая индикация:**
    *   Красные акценты для кнопок удаления.
    *   Синие акценты для кнопок восстановления.
*   **Feedback:** Использование анимированной иконки `Loader2` во время процесса восстановления, так как это требует нескольких последовательных запросов к API.
