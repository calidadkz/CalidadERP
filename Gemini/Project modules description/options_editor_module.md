# Модуль: База опций (Options Editor)

Инструмент управления глобальным справочником запчастей и узлов, которые могут выступать в роли сменяемых компонентов (опций) для станков. Модуль связывает коммерческие названия опций с реальными товарными позициями на складе.

---

## 1. Структура экрана: Реестр Групп (Левая панель)

Вертикальная панель управления категориями опций (напр. «Чиллеры», «Контроллеры»).

### Шапка управления
*   **Инструменты:**
    *   `Экспорт/Импорт`: Кнопки синхронизации всей базы опций через CSV. При импорте система автоматически создает недостающие группы и сопоставляет варианты по названиям.
    *   `+`: Кнопка быстрого создания нового типа опции (`bg-blue-600`).
*   **Форма создания группы (Inline):** Появляется вверху списка. Содержит:
    *   Поле ввода названия.
    *   Чекбоксы: «Выбор одного» (Radio) и «Обязательно» (Required).
    *   Селектор совместимости: Список категорий станков в виде тегов. Определяет, в каких конфигураторах будет доступна эта группа.

### Список групп (Registry)
*   **Карточка группы:** Интерактивный блок с белым фоном. 
    *   **Бейджи:** «Один»/«Много» и красный бейдж «Обяз.».
    *   **Теги категорий:** Список совместимых станков (напр. `#Лазерный CO2`).
    *   **Действия:** Кнопки «Карандаш» и «Корзина» появляются при наведении.

---

## 2. Основной редактор: Варианты и BOM (Правая панель)

Детальный вид выбранной группы опций, где настраиваются конкретные модели запчастей.

### Шапка раздела
*   Отображает полное название выбранной группы.
*   Акцентная кнопка **«+ Добавить вариант»** (`bg-emerald-500`).

### Форма варианта (Inline Editor)
Выдвижной блок с расширенными настройками, разделенный на две зоны:

1.  **Секция Состава (BOM):**
    *   `Название варианта`: Маркетинговое имя (напр. "Ruida 6445G").
    *   `Подбор детали`: Searchable-выпадающий список товаров типа «Запчасть».
    *   **Логика склейки:** Если деталь уже есть в составе, при повторном добавлении система не создает новую строку, а суммирует количество в существующей.
    *   `Список позиций`: Таблица привязанных запчастей с возможностью удаления (`X`).

2.  **Секция Экономики и Управления (Адаптивный блок):**
    *   `Цена закупа`: Поле ввода суммы и выбор валюты (CNY, USD и др.).
    *   **Кнопки управления:** Расположены справа. Используют динамическую ширину для предотвращения наложения. 
        *   Кнопка «Сохранить» (`bg-emerald-500`) — фиксирует изменения в БД.
        *   Кнопка «Отмена» — закрывает форму без сохранения.

---

## 3. Сетка вариантов (Gallery View)

Реестр созданных вариантов внутри группы в виде плиток (`Grid`).

### Карточка варианта:
*   **Статус:** Бейдж «Вариант» в верхнем углу.
*   **Цена:** Отображается крупным шрифтом (если разрешено правами доступа).
*   **Индикатор состава:** Плашка внизу карточки, показывающая количество позиций в BOM (напр. `2 поз.`). Если состав не заполнен, индикатор становится серым.
*   **Управление:** Быстрый переход в режим редактирования состава прямо из плитки.

---

## 4. Технические особенности UI
*   **Реактивность:** Любое изменение состава варианта мгновенно отражается на его итоговой расчетной стоимости в конфигураторе комплектаций.
*   **Адаптивность:** Форма редактирования варианта использует `flex-wrap`, что позволяет кнопкам управления «перетекать» под поля ввода на узких экранах, исключая визуальные баги.
*   **Права доступа:**
    *   `col_purchase_price`: Полностью скрывает поля ввода цен и их отображение в карточках для рядовых сотрудников.
    *   `manage_groups`: Блокирует возможность изменения структуры дерева опций.
    *   `col_bom_composition`: Скрывает детализацию складского состава запчастей.
