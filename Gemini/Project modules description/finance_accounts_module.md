# Модуль: Наши счета (Treasury / Управление ликвидностью)

Центральный пульт управления валютными резервами компании и анализа себестоимости валюты по методу FIFO.

---

## 1. Основной экран: Dashboard счетов

Визуальное представление текущей ликвидности предприятия.

### Сетка счетов (Grid)
*   **Карточки счетов:** Интерактивные блоки с балансом, названием банка и номером счета.
*   **Акцент:** При клике на карточку она выделяется синей рамкой (`ring-blue-500`), и все данные в блоке FIFO (справа) фильтруются для отображения остатков только этого счета.
*   **Инструмент «Новый счет»:** Кнопка в шапке, открывающая компактную inline-форму для добавления реквизитов.

---

## 2. Внутренние операции (Internal Transactions)

Блок управления движением средств внутри компании без участия внешних контрагентов.

### Типы операций:
1.  **Перевод:** Перемещение средств между счетами в одной валюте. Позволяет зафиксировать комиссию банка в KZT.
2.  **Обмен (Конверсия):** Конвертация средств между разными валютами (напр., KZT -> CNY). 
    *   **Авто-генерация Лота:** При выполнении обмена система автоматически создает новый входящий лот в реестре `CurrencyLots`. 
    *   **Расчет курса:** Эффективный курс закупа валюты вычисляется мгновенно на основе списанной суммы KZT.

---

## 3. Аналитика: Валютные лоты (FIFO)

Визуализация «стеков» валюты, обеспечивающая прозрачность расчета себестоимости товаров.

### Карточка лота:
*   **Прогресс-бар:** Горизонтальная линия, показывающая, какая часть лота была «потреблена» исходящими платежами.
*   **Метрики:** Дата приобретения, первоначальный объем, текущий остаток и курс закупа в KZT.
*   **Статус:** Скрытая индикация лотов с нулевым остатком для разгрузки интерфейса.

---

## 4. Визуальные стандарты UI
*   **Сплит-интерфейс:** Разделение экрана на панель операций (слева) и панель аналитики лотов (справа).
*   **Цветовые коды:** 
    *   Синие тени и рамки для активных элементов.
    *   Мягкий серый фон для неактивных лотов.
*   **Безопасность:** Кнопка «Выполнить операцию» блокируется, если сумма списания превышает доступный баланс счета-источника.
