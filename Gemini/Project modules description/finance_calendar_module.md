# Модуль: Платежный календарь (IPP - Planned Payments)

Операционный центр финансового планирования. Модуль предназначен для контроля будущих денежных потоков (Cash Flow Forecasting) и управления очередью оплат на основе заключенных договоров (ЗП/ЗК).

---

## 1. Основной экран: План оплат

Реестр плановых траншей, сгруппированный по датам исполнения.

### Шапка управления (Header)
*   **Заголовок:** «Календарь (IPP)» с иконкой `Calendar` цвета индиго.
*   **Инструменты:**
    *   **Переключатель направлений:** Фильтрация «Доходы» / «Расходы» / «Все».
    *   **Запланировать вручную:** Кнопка для создания планового платежа, не привязанного к конкретному заказу (напр. аренда, налоги).
    *   **Создать платеж:** Быстрый переход к фиксации факта оплаты.

### Таблица траншей:
*   **Дата:** Плановая дата платежа. Просроченные даты выделяются красным индикатором.
*   **Контрагент:** Наименование поставщика или клиента.
*   **Основание:** Короткий ID заказа (напр., `#AF32D1`) синим шрифтом. Клик по ID позволяет перейти к исходному документу.
*   **Статья ДДС:** Классификация движения средств (напр., «Оплата за товар», «Логистика»).
*   **Оплата / План:** Самая важная колонка. Отображает фактически выплаченную сумму относительно плановой и прогресс-бар покрытия.
*   **Статус:** Бейджи `Ожидание` (Янтарный) или `Оплачено` (Изумрудный).

---

## 2. Бизнес-логика: IPP vs IP

Модуль реализует механизм связи плановых ожиданий с банковскими выписками.

*   **Авто-планирование:** При создании заказа (ЗП/ЗК) система автоматически генерирует записи в IPP согласно графику оплат в форме заказа.
*   **Контроль исполнения:** Статус «Оплачено» устанавливается автоматически только тогда, когда сумма привязанных фактических выписок (IP) достигает суммы планового транша.
*   **Ручное планирование:** Позволяет финансовому менеджеру закладывать операционные расходы, не имеющие документа-основания в товарном блоке.

---

## 3. Взаимодействие с Разноской (Allocation)

Кнопка **«Разнести»** в строке календаря:
*   Открывает модальное окно платежа, где уже предзаполнены контрагент, сумма (остаток транша) и валюта.
*   После проведения факта система мгновенно обновляет прогресс-бар в календаре.

---

## 4. Визуальные стандарты UI
*   **Цветовая индикация направлений:**
    *   **Зеленый:** Входящие платежи (Выручка).
    *   **Красный:** Исходящие платежи (Затраты).
*   **Интерактивность:** Использование `group-hover` эффектов для кнопок действий («Разнести», «Изменить»).
*   **Типографика:** Моноширинный шрифт для сумм и ID документов для исключения ошибок визуального восприятия цифр.
*   **Пустые состояния:** При отсутствии планов на выбранный период отображается иллюстрация «Все счета закрыты».
