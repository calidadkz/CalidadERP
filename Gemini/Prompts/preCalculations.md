Роль: Системный аналитик и разработчик в Firebase Studio.
Задача: Спроектировать модуль «Предрасчет» для логистической/торговой системы.
Предрасчет делится на блока: Общие настройки партии, Список с подробностями, Упаковочный лист.
Главная учетная валюта - KZT.

Блок “Общие настройки партии”
Здесь задаются параметры, которые влияют на расчеты в “Списке с подробностями”.
Какие параметры входят:
1. Транспортные. Все транспортные расходы распределяются по Объемам позиций.
1.1 Тариф доставки по Китаю. Изменяемое поле, по умолчанию 150 - Это кол-во USD за куб.м.
1.2 Курс валюты для доставки. Изменяемое поле, по умолчанию значение из раздела Курсы валют.
1.3 Доставка Алмата-Караганда. Изменяемое поле, По умолчанию 200000, валюта всегда в KZT.
1.4 СВХ. Изменяемое поле. По умолчанию 100000. Валюта всегда в KZT
1.5 Брокер. Изменяемое поле. По умолчанию 104000. Валюта всегда в KZT
1.6 Таможенные сборы. Изменяемое поле. По умолчанию 52000. Валюта всегда в KZT


2. Налоги. У нас есть 2 варианта реализации. С компании на Общ. (общеустановленный)  и с Упр. (упрощенный)
2.1 Основные
2.1.1 НДС - изменяемое поле. по умолчанию 16%. Налог на добавленную стоимость - фигурирует в показателях. Таможенном НДС, НДС с перепродажи и в НДС итоговом (о них см. в блоке “Список с подробностями”).

2.2 Общеустановленный режим.
2.2.1 КПН20 - изменяемое поле, по умолчанию 0%. Корпоративный подоходный налог, считается 20% с чистой прибыли, но данные в предрасчете содержать только доходы и расходы связанные с этой партией. По этому у нас есть только Валовая прибыль.

2.3 Упрощенный режим.
2.3.1 КПН4 - изменяемое поле, по умолчанию 4%. Считается от выручки.
2.3.2 Наценка при перепродаже. Изменяемое поле, по умолчанию 25%. На что влияет: закупка у поставщика в Китае - по чистой стоимости - проходит на компанию на Общеустановленном режиме. Затем мы с небольшой наценкой продаем нашей другой компании - на упрощенке. В итоге уменьшаем для себя нагрузку по НДС и КПН20, но добавляем КПН4. Однако по той же стоимости, что и чистая закупка мы продать не можем - делаем наценку от чистой цены закупки у поставщика. НДС в итоге платится с этой “перепродажной” стоимости.

3. Прочее.
3.1 Бонус Отделу продаж. Считается от выручки.

Блок “Список с подробностями.”
Это список с товарами, с подробностями по расходам и доходам и другим данным.

Типы позиций: «Запчасти» (простые) и «Комплектации» (Сложные: База + набор Опций).
Логика Опций: Стоимость комплектации = Базовая цена + $\Delta$ стоимости опций (разница между выбранной и предустановленной). Опция — это функциональный узел (справочно содержит перечень запчастей).

СПРАВОЧНЫЕ ДАННЫЕ.
1. Название. Это название модели станка или запчасти.
2. Опции - применимо только к станкам - перечисленные объекты. Можно посмотреть как реализовано их отображение и логика в списке складских Остатков или в Заказах (как Клиентов так и Поставщиков).
3. Название для поставщика. Берется из номенклатуры. Для Станков по умолчанию - это объединение названии Базы и Опций для поставщика. Если у опции нет названия для поставщика (пусто или null) - то берется наше внутреннее название. Название для поставщика можно поменять на свое прямо в таблице, но если его стереть - то заново вставляется генерируемое по умолчанию.
4. Производитель
5. Код ТНВЭД.

ТРАНСПОРТНЫЕ ДАННЫЕ.
1. Кол-во товаров в данной позиции.
2. Объем. Рассчитывается по умолчанию на основе данных номенклатуры. Если добавлены данные из Упаковочного листа (по умолчанию их нет) - то на основе Упаковочного листа.
3. Чек-бокс Не учитывать габариты. По умолчанию включен у Запчастей, выключен у Станков. Можно менять статус.
4. Габариты. Комплексный параметр. По умолчанию вносятся данные из номенклатуры. При наличии данных из упаковочного листа - заполняются они.
Отображаются габариты каждой коробки.

РАСХОДЫ ТРАНСПОРТ
1. Доставка Китай
2. Доставка Алмата-Караганда
3. Доставка до клиента. Изменяемое поле, по умолчанию 0.

ПРОДАЖА,
1. Данные о заказе. Внутренний ID заказа, Клиент, №договора (если есть). Если к товару нет заказа, то пишется “Под склад”.
2. Данные о выручке. Общая выручка. Задается из заказа. Если задаем вручную - то это планируемая цена продажи, по умолчанию ставится из номенклатуры. Важно разделить визуально когда выручка подтвержденная (на основе заказа), а когда расчетная (из номенклатуры).
3. Данные о предоплате. Аналогичны выручке, но только из Заказа - ставится цена оплаченная. Если нет проведенных оплат по заказу, или если мы заказываем под склад - то можно внести свою сумму. Она должна также отличаться от подтвержденной предоплате.

НАЛОГИ
1. Налоговый режим. Выбирает для каждой позиции, а не у всего Предрасчета! Выбираем 1 из 2 вариантов: Общ. (общеустановленный) и Упр. (упрощенный) По умолчанию ставится Общ.
2. НДС. Включает несколько значений:
2.1 НДС итоговый. Для Общ. - от выручки. Для Упр. - то цены перепродажи.
2.2 НДС таможенный считается как % НДС (в Настройках) от суммы: Чистая стоимость товара + 60% доставки Китай.
2.3 НДС разница. Разность итогового и таможенного. 
5. КПН - считается по-разному для Общ. и Упр. режимов. Подпись в интерфейсе меняется КПН4 или КПН20 - в зависимости от режима.

ФИНАНСЫ ОБЩЕЕ
1. Прибыль. Итоговая - после всех расходов.
2. Рентабельность. Изменяемое поле - остальные рассчитываемые данные подстраиваются. По умолчанию стоит значение на основе продажной цен из номенклатуры, либо из заказа (если есть связанный заказ)
3. Себестоимость полная
4. Себестоимость допродажная: НЕ включены Бонусы ОП, доставка до клиента, пусконаладка, КПНы. НДС учитывается по-разному: для Общ. - только НДС таможенный, для Упр. - НДС итоговый.

ПРОЧЕЕ
1. Бонус Отделу Продаж (ОП)
2. Пусконаладка

Блок “Упаковочный лист”.
Блок будет расписан подробнее в следующем промпте. Общие сведения для контекста:
Представляет собой интерактивную панель - в которой надо разместить товары по коробкам.
Экран поделен на 2 половины. Слева список с товарами. Справа список с коробками и их содержимым.
В итоге необходимо распределить все содержимое по коробкам - на основе этих данных пересчитываются габариты из расчетных (номенклатурных) на реальные.
